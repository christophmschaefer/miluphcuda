1 Giant Collision examples for miluphcuda
2 ---------------------------------------
3 
4 Christoph Burger
5 christoph.burger@uni-tuebingen.de
6 
7 last updated: 22/Feb/2021
8 
9 -----------------------------------------
10 
11 These examples represent a collision of a ~1/10 Moon-mass body with a ~Moon-mass target.
12 Both bodies have a 30 mass% iron core and a 70 mass% rocky mantle.
13 The collision velocity is two times the mutual escape velocity (at the moment of first contact).
14 The impact angle is 45 deg.
15 
16 You can choose between two material rheologies:
17 
18 * *hydro* -- simple strengthless bodies
19 * *solid* -- includes solid-body physics, the Grady-Kipp fragmentation model, and the Collins plasticity model
20 
21 The Tillotson EoS is applied to model the thermodynamic response.
22 
23 The scenarios use ~60k SPH particles, with runtimes around 1.5h for *hydro*,
24 and 3h for *solid* on most current GPUs (benchmarked on a GTX 970).
25 
26 -----------------------------------------
27 
28 **To run the examples:**
29 
30 1. Compile miluphcuda using the `parameter.h` file from the respective directory (hydro or solid).
31  Don't forget to also adapt the Makefile to your system.
32 2. Unpack `impact.0000.gz`.
33 3. Adapt the start script `run.sh` to your system (path to CUDA libs and to miluphcuda executable) and execute it.
34 4. Wait for the simulation to finish (75 output files).
35  Output to stdout and stderr is written to `miluphcuda.output` and `miluphcuda.error`, respectively.
36 
37 -----------------------------------------
38 
39 **Check/visualize the results**
40 
41 * You can visualize the simulation for example with *Paraview*. Find the latest release at https://www.paraview.org/.
42 
43  First, enter the simulation directory and run
44 
45  utils/postprocessing/create_xdmf.py
46  which creates an xdmf file (*paraview.xdmf* by default). You can use the default cmd-line options,
47  so no need to set anything explicitly. Then start Paraview and either
48 
49  * directly open the created paraview.xdmf and choose settings yourself
50  * load the prepared Paraview state in *paraview.pvsm* (*File -> Load State*), and select
51  the created paraview.xdmf file under *Choose File Names*
52 
53 * Compare your results (e.g., visualized with Paraview) to the animations in the `results/` directories,
54  which show the interior structure of the colliding bodies (cut views) for various quantities.
55 
56 -----------------------------------------
57 
58 **Where to go from here?**
59 
60 You can easily build on those examples for setting up your own simulations. It is straight-forward to run different scenarios,
61 with varying masses, collision parameters, and even material compositions, and pre-collision rotation. We use an external
62 tool for creating the required initial conditions, which is easy to use. Just drop me an e-mail if you are interested.
63 
64 You can also run further postprocessing on the results, e.g., to find all collision fragments and
65 gravitationally bound aggregates of those. The necessary tools are included in
66 
67  miluphcuda/utils/postprocessing/fast_identify_fragments_and_calc_aggregates/
68 
69 Check the QuickStart guide there for more details.
70 
71 Enjoy!
