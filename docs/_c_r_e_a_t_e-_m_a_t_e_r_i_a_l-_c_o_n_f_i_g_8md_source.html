10 
11 Purpose
12 -------
13 
14 `miluphcuda` supports simulations comprising multiple materials, and multiple rheologies.
15 Many important settings for the material model and numerics, and especially material parameters,
16 are provided in a structured config file, processed by the _libconfig_ library.
17 
18 Check http://hyperrealm.github.io/libconfig/ for the latest version.
19 
20 Note: Important settings for `miluphcuda` are also set via cmd-line flags, and at compile time in `parameter.h`.
21 
22 --------------------------------
23 
24 File structure
25 --------------
26 
27 The config file (typically named `material.cfg`) consists of key-value pairs,
28 organized in groups for different materials, equation of state, etc.
29 
30 The basic structure of the material config file, containing `n` different materials, looks like:
31 
32  materials =
33  (
34  {
35  ID = 0
36  name = ...
37  ...
38  },
39  {
40  ID = 1
41  name = ...
42  ...
43  },
44  .
45  .
46  .
47  {
48  ID = n-1
49  name = ...
50  ...
51  }
52  );
53 
54 The `material.cfg` file in this directory contains an example setup, comprising several materials and material models.
55 
56 --------------------------------
57 
58 Where to find material parameters?
59 ----------------------------------
60 
61 * The directory `material_data/` contains parameters for various materials.
62 
63 * You can also check the `test_cases/` for example material setups.
64 
65 --------------------------------
66 
67 Good to know...
68 ----------------------------------
69 
70 The entries in the config file have to match their data type. For example, the ID tag has
71 to be an integer value whereas the value for the cohesion has to be given in floating point
72 notation:
73 
74  cohesion = 1000
75 
76 will not work but
77 
78  cohesion = 1000.0
79 
80 or
81 
82  cohesion = 1e3
83 
84 are valid entries.
85 
86 The code prints out all read material parameters if the verbose flag `-v` is used.
87 If values are ommitted default values are assumed. Normally, these default values are not what you
88 want :ghost:.
