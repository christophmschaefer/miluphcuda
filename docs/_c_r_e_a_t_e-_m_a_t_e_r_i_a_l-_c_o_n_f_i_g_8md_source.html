2 =====================================================

9 --------------------------------
10 
11 Purpose
12 -------
13 
14 miluphcuda supports multiple materials, and multiple rheologies. Many important settings for
15 the material model, especially material parameters, are parsed in a structured config file,
16 typically named *material.cfg*. In addition to material parameters, some numerical settings
17 (e.g., for the smoothing length) are also configured via that file.
18 
19 The config file is processed by the *libconfig* library.
20 Check http://hyperrealm.github.io/libconfig/ for the latest version and documentation.
21 
22 **Note**: This file is one of three places where you configure miluphcuda settings. The other two are:
23 
24 * *parameter.h* (at compile time)
25 * *cmd-line flags*
26 
27 The config file is passed to miluphcuda by the `-m` cmd-line option.
28 
29 --------------------------------
30 
31 File structure
32 --------------
33 
34 The config file consists of key-value pairs, organized in groups for different materials,
35 equation of state, etc. Groups are blocks enclosed in `{...}`.
36 
37 The basic file structure, containing *n* different materials, looks like:
38 
39  materials =
40  (
41  {
42  ID = 0
43  name = ...
44  ...
45 
46  eos = {
47  type = ...
48  ...
49  ...
50  }
51  },
52  {
53  ID = 1
54  name = ...
55  ...
56 
57  eos = {
58  type = ...
59  ...
60  ...
61  }
62  },
63  .
64  .
65  .
66  {
67  ID = n-1
68  name = ...
69  ...
70 
71  eos = {
72  type = ...
73  ...
74  ...
75  }
76  }
77  );
78 
79 The available/required settings depend on the used material model(s), and are listed and described below.
80 
81 The `material.cfg` file in this directory contains an example setup, comprising several materials and material models.
82 
83 Note that, for now, the `eos = {...}` group regularly contains not only equation-of-state-specific entries,
84 but also further parameters related to the material model.
85 
86 --------------------------------
87 
88 Where to find material parameters?
89 ----------------------------------
90 
91 * The `material_data/` directory contains parameters for various materials.
92 
93 * You can also check the `test_cases/` directory for example material setups.
94 
95 --------------------------------
96 
97 Good to know...
98 ---------------
99 
100 * The entries in the config file have to match their data type. For example, the ID tag has to be
101  an integer value whereas the value for the cohesion has to be given in floating point notation:
102 
103  cohesion = 1000
104 
105  will not work but
106 
107  cohesion = 1000.0 or cohesion = 1.0e3
108 
109  are valid entries. The libconfig specs indicate that a period is *always* mandatory for floats...
110 
111 * The code prints out all read material parameters if the verbose flag `-v` is used.
112 
113 * If values are ommitted, default values are assumed (see below). Normally, these default values are not what you want.
114 
115 --------------------------------
116 
117 Available settings
118 ------------------
119 
120 Here, all settings are described, ordered mainly by material model and/or equation of state.
121 For the keys in the following lists, the top level is assumed to be a specific material and
122 subgroups within materials are denoted by dot syntax. For example, *ID* is on the top level
123 of some material, and *eos.type* is the key *type* in the subgroup *eos*.
124 
125 This list is currently not exhaustive.
126 
127 
128 **Always mandatory**:
129 
130  Key Type Default Details
131  _______________________________________
132 
133  ID int none material ID, has to fit SPH particles' material-type
134  in input file, IDs have to be 0, 1, 2,...
135  eos.type int none set equation of state for material, for a complete list see
136  https://christophmschaefer.github.io/miluphcuda/pressure_8h.html
137  or look directly into pressure.h
138 
139 **Always optional**:
140 
141  Key Type Default Details
142  ___________________________________
143 
144  name str none name of material, currently not processed by miluphcuda
145 
146 **Tillotson EoS**:
147 
148 To use the Tillotson EoS for some material, set *eos.type = 2*, and the following parameters.
149 
150  Key Type Default Details
151  _______________________________________________________
152 
153  eos.till_rho_0 float 0. reference density
154  eos.till_A float 0.
155  eos.till_B float 0.
156  eos.till_E_0 float 0.
157  eos.till_E_iv float 0.
158  eos.till_E_cv float 0.
159  eos.till_a float 0.
160  eos.till_b float 0.
161  eos.till_alpha float 0.
162  eos.till_beta float 0.
163  eos.rho_limit float 0.9 for expanded states with rho/rho_0 < rho_limit
164  the pressure is set to zero (if e < till_E_cv)
165  eos.cs_limit float 1% of approx. lower limit for sound speed (in m/s),
166  bulk sound speed default is 1% of sqrt(till_A/till_rho_0)
167 
168 **P-alpha porosity**:
169 
170 The P-alpha porosity model can be used together with several EoS, which all have a distinct *eos.type*
171 when used together with the P-alpha model:
172 
173 * *eos.type = 5* ... P-alpha + Tillotson EoS
174 * *eos.type = 6* ... P-alpha + Murnaghan EoS
175 * *eos.type = 13* ... P-alpha + ANEOS
176 
177 in addition to the parameters for the respective EoS, you need the following:
178 
179  Key Type Default Details
180  _______________________________________________________
181 
182  ...
183  ...
184  ...
185  eos.cs_porous float 50% of approx. sound speed in uncompacted material (in m/s),
186  bulk sound speed the actual sound speed is interpolation between
187  this and the matrix cs, based on distention
