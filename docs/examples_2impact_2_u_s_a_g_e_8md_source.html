1 Impact example for miluphcuda
2 ---------------------------------------
3 
4 Christoph Burger
5 christoph.burger@uni-tuebingen.de
6 
7 last updated: 08/Jun/2021
8 
9 ----------------------------------------------------------------------------------
10 
11 
12 This example represent an impact scenario of a basalt projectile onto a basalt, half-sphere target.
13 
14 The spherical projectile has a radius of 0.5m, impacts with 6km/s, at an impact angle (to the vertical) of 30 deg.
15 
16 All material is modeled as a porous solid, with a porosity of 20% (i.e., an initial distention of 1.25).
17 
18 Applied rheology models:
19 
20 * p-alpha porosity model
21 * Collins model for plastic flow
22 * Grady-Kipp model for fragmentation
23 * Tillotson equation of state
24 
25 A zero-gravity environment is assumed.
26 
27 The scenario uses ~60k SPH particles, with a runtime on the order of one hour on most current GPUs (benchmarked on a GTX 970).
28 
29 ----------------------------------------------------------------------------------
30 
31 
32 **To run the example:**
33 
34 1. Compile miluphcuda using the `parameter.h` file from this directory.
35  Don't forget to also adapt the miluphcuda Makefile to your system.
36 2. Unpack `initials/impact.0000.gz` and move `impact.0000` to this directory.
37 3. Adapt the start script `run.sh` to your system (path to CUDA libs and to miluphcuda executable) and execute it.
38 4. Wait for the simulation to finish (200 output files).
39  Output to stdout and stderr is written to `miluphcuda.output` and `miluphcuda.error`, respectively.

44 **Check/visualize the results**
45 
46 * You can visualize the simulation for example with *Paraview*. Find the latest release at https://www.paraview.org/.
47 
48  First, enter the simulation directory and run
49 
50  utils/postprocessing/create_xdmf.py
51 
52  which creates an xdmf file (*paraview.xdmf* by default). You can use the default cmd-line options, so no need to set
53  anything explicitly. Note that the xdmf file contains only metadata, i.e., you still need the .h5 files it points to
54  in the same directory.
55  Then start Paraview and either
56 
57  * directly open the created xdmf file and choose settings yourself
58  * load the prepared Paraview state in `results/paraview.pvsm` (*File -> Load State*), and select
59  the created paraview.xdmf file under *Choose File Names*
60  (note that this shows you a slice of the target by default, for better visibility of the cratering process)
61 
62 * Compare your results (e.g., visualized with Paraview) to the animations in `results/animations/`,
63  which show the cratering process (a sclice of the target) for various quantities.
64 
65 * A full view of the (half-sphere) target and the impact process is visualized in `results/animations/full.mp4`.
66 
67 * An important aspect of the porosity model is that the *crush curve* (the curve distention vs. pressure which is
68  followed during compaction) is modeled correctly. Since the pressures that lead to compaction are typically released
69  again quickly, the SPH particles will eventually lie either on or below the crush curve -- if the model is accurate.
70  You can check this by running for example
71 
72  results/plot_p_alpha_convergence.py -v --files impact.000[1-4].h5
73 
74  which plots SPH particles + the theoretical crush curve for output frames 1 through 4. Take a look at
75  `results/p_vs_alpha.first4frames.png` to see what the plots should qualitatively look like for the above four frames.
76  You can of course specify any number/sequence of frames you are interested in for plotting.
77 
78 * You can also visualize the workings of the Collins plasticity model by running `results/plot_plastic_yielding.sh`.
79  This produces plots for shear stress vs. pressure, including the respective yield limit curves.

84 **Where to go from here?**
85 
86 You can easily build on this example for setting up your own simulations. It is straight-forward to run different
87 scenarios, with varying impactor masses, impact parameters, and even material compositions and parameters.
88 We use an external tool for creating the required initial conditions, which is easy to use.
89 Just drop me an e-mail if you are interested.
90 
91 Take a look at the timestep statistics at the very bottom of *miluphcuda.output*. If you are not satisfied you may try
92 to adjust the integrator accuracy on the cmd-line and/or the compile-time settings for the integrator in *rk2adaptive.h*.
93 
94 Enjoy!
